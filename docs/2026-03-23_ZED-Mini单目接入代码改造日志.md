# 2026-03-23 ZED-Mini 单目接入代码改造日志

## 目标

根据本地 zed-sdk 与接入指南，完成 Camera 侧 ZED-Mini 正确初始化和工厂标定读取，并在 Vision 中应用左目参数，实现“先单目左目可用”的稳定链路。

## 本次改动文件

1. src/controller/player/sensor/camera.hpp
2. src/controller/player/sensor/camera.cpp
3. src/controller/player/vision/vision.hpp
4. src/controller/player/vision/vision.cpp

## 关键改动说明

1. Camera 增加 ZED 状态与标定对外接口
- 新增 use_zed()。
- 新增 get_zed_left_camera_param(camera_param &para)。
- 新增 ZED 标定缓存成员（左目内参与畸变、FOV、双目基线 tx、标定分辨率）。

2. Camera::open 中在 zed.open 成功后读取工厂标定
- 读取路径：getCameraInformation().camera_configuration.calibration_parameters。
- 提取左目参数：fx/fy/cx/cy、disto[0..3] 对应 k1/k2/p1/p2。
- 提取辅助信息：h_fov/v_fov/d_fov、stereo_transform 的 tx。
- 按当前输出分辨率（image.width/image.height）做内参缩放，保证与实际输出图尺寸一致。
- 打印初始化日志，便于在线核对参数。

3. Vision 首帧接入 ZED 时应用工厂标定
- 在 updata(SENSOR_CAMERA) 中，当检测到 ZED 后端且未应用过标定时：
  - 用 Camera 提供的左目标定覆盖 params_。
  - 同步更新 camK、D、invCamK。
  - 把新参数重新 cudaMemcpy 到设备侧缓存（pCamKData/pInvNewCamKData/pDistortData）。
- 记录一次性日志，确认参数已生效。

4. 运行语义保持
- 保持当前“MV 失败 -> ZED -> V4L2”回退链。
- 保持先将 ZED 当单目左目使用，不改 detector 接口与上层行为逻辑。
