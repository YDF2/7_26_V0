# 2026-03-23 阶段A最小修复：ZED 路径跳过去畸变实施记录

## 目标

按 Qt 黑屏方案中的“阶段 A 最小修复”落代码：仅在 ZED 路径先跳过去畸变，快速恢复 debuger 画面可见；同时保持检测结果画框链路可用。

## 修改文件

1. src/controller/player/vision/vision.hpp
2. src/controller/player/vision/vision.cpp

## 具体改动

1. 增加 ZED 后端状态位
- 在 Vision 成员中新增 `use_zed_`。
- 在首次接收相机数据时，从 Camera 读取并保存 `use_zed_ = sptr->use_zed()`。

2. 调整图像预处理分支（核心）
- 保持 MV 路径不变：继续执行 cudaUndistored。
- ZED 路径改为阶段 A 临时策略：
  - 直接 `cudaMemcpy(dev_ori_ -> dev_undis_)`。
  - 不执行 cudaUndistored。
- V4L2 路径保持原逻辑（不去畸变）。

3. 增加一次性运行提示日志
- 当检测到 use_zed_ 时，打印：
  - Vision: Stage-A ZED path bypasses undistortion to recover debug image visibility.

## 为什么这能恢复 Qt 画面

当前 Qt ImageMonitor 显示的是 Vision 发送的 JPG 图。此前黑屏高概率来自 ZED+remap 去畸变阶段的大面积无效映射。阶段 A 直接绕过去畸变，只保留 BGR->resize->检测->编码发送链路，可快速恢复可见画面。

## 检测框是否保留

保留。

原因：

1. 检测输入仍来自 `dev_undis_`（现在是 ZED 原图 resize 后直通版本）。
2. IMAGE_SEND_RESULT 分支中的球/球门框绘制逻辑未改。
3. `send_image()` 的 JPG 发送逻辑未改。

## 风险与后续

1. 阶段 A 是“可视化恢复优先”，暂时牺牲了 ZED 去畸变。
2. 后续应进入阶段 B：统一分辨率宽高比并重建几何一致性，再逐步恢复 ZED 去畸变。
