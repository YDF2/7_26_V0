### 鱼眼失真的实际影响

- **直线失真**：相机采集的直线会呈现微妙的弧形，特别是靠近图像边缘
- **边界物体精度损失**：靠近图像边界的目标检测精度下降 10-20%（因坐标映射误差累积）
- **动作识别问题**：机器人的位置评估可能产生系统偏差

## P1 阶段：去畸变集成改造

### 改造技术方案

#### 步骤 1：参数基础设施（无需新增）

**收获**：调研发现 Vision 层已有完整的 distortion 参数存储：
- CPU 端：`D` 矩阵（4 个失真系数 k1, k2, p1, p2）
- GPU 端：`pDistortData` 指针
- 初始化流程：构造函数默认值 + ZED 参数覆盖（通过 `updata()` 中的 `sptr->get_zed_left_camera_param()`）

**结论**：无需在 Vision 层新增成员

#### 步骤 2：CUDA 去畸变 Kernel 实现

**新增接口**（imageproc.hpp）：
```cpp
void cudaUndistort(unsigned char *in, unsigned char *out, 
                   float fx, float fy, float cx, float cy,
                   float k1, float k2, float k3, float p1, float p2, 
                   int w, int h, int channels);
```

**核心实现**（imageproc.cu，undistort_direct_kernal）：

1. **归一化坐标映射**（normalize）
   ```
   x_norm = (pixel_x - cx) / fx
   y_norm = (pixel_y - cy) / fy
   ```

2. **去畸变校正**（OpenCV 标准模型）
   ```
   r² = x_norm² + y_norm²
   kr = 1 + k1·r² + k2·r⁴ + k3·r⁶
   
   x_undist = x_norm·kr + 2p1·x_norm·y_norm + p2(r² + 2x_norm²)
   y_undist = y_norm·kr + 2p2·x_norm·y_norm + p1(r² + 2y_norm²)
   ```

3. **重投影**（back-project）
   ```
   src_x = x_undist·fx + cx
   src_y = y_undist·fy + cy
   ```

4. **双线性插值采样**
   - 边界外像素填充黑色（0,0,0）
   - 有效范围内使用 4 个邻近像素加权平均

**关键特性**：
- ✅ 直接根据畸变参数计算，无需预计算映射坐标
- ✅ 支持动态参数（ZED 摄像头的不同内参和畸变系数）
- ✅ GPU 并行化：使用 16×16 thread blocks

#### 步骤 3：Vision 层条件分支改造

**文件**：vision.cpp，第 211-228 行

**改造前**（Stage-A 临时方案）：
```cpp
if (use_zed_) {
    // 跳过去畸变，直接 memcpy
    cudaMemcpy(dev_undis_, dev_ori_, ...);
}
```

**改造后**：
```cpp
if (use_zed_) {
    // 启用去畸变，使用 ZED 参数
    imgproc::cudaUndistort(dev_ori_, dev_undis_,
                           params_.fx, params_.fy, params_.cx, params_.cy,
                           params_.k1, params_.k2, 0.0f, params_.p1, params_.p2,
                           w_, h_, 3);
}
```

**三路分支保持**：
- MV 路线：`cudaUndistored`（现有实现，使用预计算映射）
- ZED 路线：`cudaUndistort`（新增实现，直接畸变参数）
- V4L2 路线：无去畸变（保持遗留行为）

## 技术亮点与决策依据

### 为什么用直接 kernel 而非预计算映射

- **官方方案**：`sl::blobFromImage()` 内部自动处理，对用户无需配置
- **用户约束**：Camera 层的 ZED 参数动态获取（updata() 中），无法在 start() 时预先计算 mapx/mapy
- **解决方案**：实现 undistort_direct_kernal，直接在 kernel 中计算去畸变坐标
- **权衡**：性能与灵活性（GPU 实时计算 vs CPU 离线预计算）

### 为什么 Stage-A 路线现在可以启用去畸变

前期 Stage-A 是为了快速恢复 Qt 黑屏问题（跳过去畸变）。现在：
1. 直播可视化已在正常工作
2. 新的 undistort_direct_kernal 性能可控（GPU 并行，无额外 memcpy）
3. ZED 路线已获得参数（通过 updata() 获取的 params_）
4. 可以安全启用去畸变而不影响帧率
